## 二、布局结构简要说明
整个界面采用 Jetpack Compose 声明式 UI 编写，使用组件化思想将界面拆分为多个可组合函数，布局清晰、层次分明。

1. BusinessCardApp（根布局）
- 使用 Column 作为根容器
- 设置全屏大小、自定义钢蓝色背景
- 整体内容水平居中、垂直居中
- 界面分为上半部分 CardTop 和下半部分 CardBottom

2. CardTop（上半部分信息区）
- 使用 Column 垂直排列
- 包含头像、姓名、职位三个组件
- 内容水平居中，文字样式区分明显

3. CardBottom（下半部分联系方式区）
- 使用 Column 纵向排列
- 包含三行联系方式，每行使用 ContactRow 组件
- 布局整齐、样式统一

4. ContactRow（单行联系方式）
- 使用 Row 横向排列
- 左侧为 Material 图标，右侧为文字信息
- 图标与文字垂直居中对齐，视觉规范统一

## 三、遇到的问题和解决过程
1. 颜色直接写死不利于维护问题
问题：直接在 background 里写十六进制颜色，代码不规范、不易修改
解决：将背景色、文字色、图标色定义为单独颜色变量，符合实验要求且便于统一管理

2. 联系方式排版不整齐问题
问题：图标和文字没有对齐，三行联系方式显得杂乱
解决：在 Row 中设置 verticalAlignment = Alignment.CenterVertically，实现居中对齐

3. 界面内容拥挤问题
问题：头像、姓名、联系方式之间间距过小，界面拥挤
解决：使用 Spacer 组件设置合理间距，让层次更清晰、视觉更舒适

4. 电话号码字体偏小问题
问题：联系方式文字太小，查看不清晰
解决：增大联系方式字体大小，让关键信息更醒目
